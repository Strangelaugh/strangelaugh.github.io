Projects I'm interested in:


    
    
      
  • 
        
    
          
    Functional implementation of the Trusted
      Platform Module (preferably in OCaml/Coq or F#/F*)
    
          
    The TPM specification is written in (sometimes bad) C
      code.
    
          
    Functional languages are considered safer in general and
      often produce produce less buggy code.
    
          
    At least the TPM specification should be as close to bug
      free as possible.
    
          
    Virtual TPMs are often used as virtual security modules
      and would greatly benefit from a secure implementation even if
      hardware vendors ultimately need C code.
    
          
    A functional specification would make it far more
      amenable to formal verification than the current one.
    
        
    
      
    • 
      
  • 
      
    
      
    ... a verifiable model and specification of the TPM.
    
      
    Operating systems, compilers, and entire distributed
      systems are formally verified--yet the TPM
      isn't...
    
      
    Inspired by current RISC-V work, developing a practical
      verification framework for TPM would be challenging, interesting, and
      could uncover a variety of bugs.
    
      
    This would also allow meaningful simulation and evaluation of trusted computing protocols in the evolving virtualization space.
    
      
    
      
    • 
      
  • 
      
    
      
    Implementing virtual TPMs using MirageOS
    
      
    Unikernels on Xen provide a much smaller TCB compared to
      solutions implementing vTPMs as processes of a full OS.
    
      
    Current research shows that such micro VMs can be launched
      with fork-like speeds.
    
      
    Naturally improved and hardware supported isolation compared
      to vTPMs as processes on anything that isn't a microkernel.
    
      
    
      
    • 
      
  • 
      
    
      
    Verifying complex (and trusted) systems on an architecture
      level at scale (Bigraphs are particularly interesting).
    
      
    Formal verification remains notoriously time consuming and
      expensive.
    
      
    Most trusted computing systems can be specified on an
      architecture level (does not mean that code verification can be
      skipped).
    
      
    Verifying security properties of trusted computing systems
      aids system design and guides verification effort and can be
      less expensive.
    
      
    Many interesting frameworks available, no clear winners yet.
    
      
    
      
    • 
      
  • 
      
    
      
    TPM out of nothing (or with less...)
    
      
    Secure co-processors and well isolated execution
      environments are provably secure against a variety of software/hardware attacks.
    
      
    Extra hardware can be expensive, slow, and unavailable.
    
      
    The goal is to implement equivalent functionality while reducing the amount of extra hardware need.
